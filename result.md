# 01. 프로젝트 개요

## 프로젝트 주제 및 선정 배경

### 주제
> **"삽가능 스튜디오" - AI 기반 광고 콘텐츠 자동 생성 SaaS 플랫폼**

소상공인과 마케터들이 **상품 이미지와 상품명만으로** 전문적인 광고 영상과 이미지를 자동으로 생성할 수 있는 서비스입니다.

### 선정 배경

| 문제점 | 해결 방안 |
|--------|----------|
| 소상공인의 광고 콘텐츠 제작 비용 부담 | AI 자동화로 제작 비용 대폭 절감 |
| 전문 디자이너/영상 제작자 고용 어려움 | 누구나 쉽게 사용 가능한 UI/UX |
| 광고 문구 작성의 어려움 | AI가 5가지 광고 문구 자동 생성 |
| 영상 제작 시간 소요 (최소 수일~수주) | 수 분 내 자동 생성 완료 |

### 기획 의도

1. **접근성** - 기술적 지식 없이도 누구나 광고 콘텐츠 제작 가능
2. **효율성** - AI 자동화로 제작 시간과 비용 최소화
3. **품질** - 전문가 수준의 광고 영상/이미지 품질 보장
4. **확장성** - 스토리보드 기능으로 복잡한 영상도 제작 가능

---

## 프로젝트 내용

### 핵심 기능

```
1. 광고 영상 자동 생성
   - 상품 이미지 업로드 → AI 광고문구 5개 생성 → 영상 자동 제작
   - 크레딧 비용: 80크레딧/건

2. 광고 이미지 자동 생성
   - 상품 이미지 기반 광고 이미지 자동 제작
   - 크레딧 비용: 20크레딧/건

3. AI 스토리보드 (고급 기능)
   - AI가 씬 구성 초안 자동 생성
   - 씬별 이미지/클립 생성 후 최종 영상 병합
   - 세부적인 영상 컨트롤 가능

4. 크레딧 시스템
   - TossPayments 연동 결제
   - 신규 가입 시 20크레딧 무료 제공
   - 사용량 기반 과금 체계

5. 대시보드
   - 생성한 콘텐츠 관리
   - 실시간 생성 진행률 모니터링
   - 다운로드/공유/삭제 기능
```

### 컨셉
> **"이미지 한 장으로 광고 영상 완성"**

복잡한 영상 편집 툴 없이, 단 4단계 만에 전문적인 광고 콘텐츠를 생성합니다.

# 04. 프로젝트 수행 경과

## 1. 핵심 기능 구현 - 영상 생성 워크플로우

### 4단계 생성 프로세스

```
Step 1: 이미지 업로드
├─ Drag & Drop 지원
├─ 이미지 미리보기
├─ Supabase Storage 저장
└─ 파일 검증 (10MB 이하, jpg/png/webp)

Step 2: 상품 정보 입력
├─ 상품명 (필수)
├─ 상품 설명 (선택)
└─ 실시간 유효성 검사

Step 3: 광고 문구 선택
├─ AI가 5개 광고 문구 자동 생성
├─ 카드 형태로 미리보기
└─ 클릭으로 선택

Step 4: 영상 생성
├─ n8n 워크플로우 트리거
├─ 실시간 진행률 표시
├─ 완료 시 자동 리다이렉트
└─ 크레딧 80 차감
```

> Step 3이 핵심인데요, **n8n 워크플로우**가 호출되어 AI가 광고 문구 5개를 자동 생성합니다. 사용자는 마음에 드는 문구를 클릭해서 선택하면 됩니다.
>
> Step 4에서 생성 버튼을 누르면 80크레딧이 차감되고, 영상 생성이 시작됩니다. **Supabase Realtime**으로 진행률을 실시간 모니터링할 수 있습니다.

## Q&A 예상 질문

### Q1: n8n 워크플로우는 어떻게 구성되어 있나요?
> n8n에서 웹훅을 받으면, 상품 정보를 기반으로 AI 모델을 호출해서 광고 문구를 생성합니다. 그 다음 영상 생성 API를 호출하고, 완료되면 Supabase 테이블을 업데이트하는 방식입니다.

### Q2: 영상 생성은 어떤 AI 모델을 사용하나요?
> n8n 워크플로우 내에서 외부 AI 서비스 API를 호출하는 방식입니다. 구체적인 모델은 비공개이지만, 영상 생성에 특화된 서비스를 연동했습니다.

### Q3: 크레딧이 부족하면 어떻게 되나요?
> 크레딧 부족 모달이 표시되고, 결제 페이지로 안내됩니다. 결제 완료 시 즉시 크레딧이 충전되어 이어서 생성할 수 있습니다.

### Q4: 개발 중 가장 어려웠던 점은?
> Clerk과 Supabase의 인증 연동이 까다로웠습니다. 2025년 4월 이후 네이티브 통합 방식이 권장되는데, 관련 문서가 적어서 시행착오가 있었습니다. 결국 Clerk 토큰을 Supabase에 전달하는 방식으로 해결했습니다.
